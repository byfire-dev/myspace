THE LOOP
从"再做一次"到"以后都更容易"

关键不在于任务做得更快，而在每次完成之后，都把判断回写成下一次默认可用的系统能力。

1
Plan

先看材料、旧规则、旧案例。不是空白提示，而是带着历史上下文开工。

2
Work

让 AI 先生成初稿、分析、回复和结构化结论，但执行仍由人来压低。

3
Review

人类介入的价值不是补字，而是判断哪些错误会重复出现、哪些标准值得固化。

4
Compound

把教训回写进模板、清单、术语表和门槛，下一次就不必从头解释。

CORE SHIFT
沉淀下来的到底是什么

真正被复用的不是这次答案本身，而是标准、记忆、路由和评估这些系统资产。

答案会过期，但判断依据、失败样例、升级条件和验收口径会持续复用。

Standards

什么算合格、什么不该出现、口径如何统一。

Memory

历史案例、反例、术语表和已经踩过的坑。

Routing

什么情况自动通过、什么情况升级、拦截或转人工。

Evaluation

用什么检查系统有没有更准，而不是只看感觉。

WRONG FRAME
为什么很多人越用 AI 越累

因为工作流是线性的：写完就发，答完就算，分析完就结束。每次任务都像一块孤岛。

DON'T JUST SHIP OUTPUT
LEVERAGE
真正高杠杆的部分

在 agent 速度足够快之后，瓶颈往往不再是"做"，而是"做什么"与"学到了什么"。

PLAN + REVIEW WORK

高价值工作量合并 planning 与 review 偏移，因为那里决定系统是否会越来越准。

APPLIES TO

LINEAR VS COMPOUND
两种系统的差别

真正的差别不是快 20%，而是第 N 次任务会不会自动携带前 N-1 次的判断沉淀。

Linear

第 20 次仍像第 1 次。

Compound

第 20 次会调用前 19 次积累。
