OpenCode 把 agent 能力下沉为一个带类型契约的本地服务,用 OpenAPI 把客户端与内核彻底解耦——这是它区别于「TUI+模型调用捆死在一个进程」的单体编码工具的根本。下面是六大核心理念。

🔌客户端/服务端解耦

opencode 同时拉起无头 HTTP 服务端 + TUI 客户端;任何客户端都经同一套 HTTP API 交互。

📐API 优先 · SDK 自动生成

OpenAPI 3.1 规格即真相来源,@opencode-ai/sdk 由规格生成,类型端到端一致。

🧩模型无关 · 无锁定

AI SDK + Models.dev,75+ provider,provider/model 统一 ID,/connect 随时切换。

🔒可审计权限

allow/ask/deny 细粒度;bash 命令经 tree-sitter 解析后按模式审批。

↩️Git 原生可回退

影子 git 仓库做快照,/undo · /redo 逐条回退 agent 的文件改动。

⌨️本地优先 · 可编程

默认只听 127.0.0.1:4096;既驱动 TUI,也可被脚本/CI/IDE 当后端编排。

内置主 agent(Tab 切换):build(默认,全权限)、plan(只读分析,edit/bash 默认 ask/deny);另有 general / explore / scout 等子代理用于并行多步任务。
01

客户端/服务端架构 + OpenAPI

理解 OpenCode 的钥匙:运行 opencode 即在单个 Bun 进程内拉起一个无头服务端和一个 TUI 客户端,服务端发布 OpenAPI 规格,众多客户端经同一套 HTTP API 接入。

02

Monorepo 包结构与技术栈

Bun workspaces + Turbo 的 monorepo,版本集中在 catalog。核心能力都在 packages/opencode(npm 发布名 opencode-ai)。

包职责
packages/opencode核心:CLI、无头服务端、主循环、工具、provider、LSP/MCP 客户端、会话存储
packages/sdk/js由 OpenAPI 生成的 TypeScript SDK(@opencode-ai/sdk,含 types.gen.ts)
packages/plugin插件 API(@opencode-ai/plugin),导出 Plugin 类型与 tool() 帮助函数
packages/app / webWeb 客户端(SolidStart) / 营销+文档站(Astro Starlight)
packages/desktop桌面应用(Tauri)
packages/console/*云端(Share/Auth/计费):Cloudflare Workers + S3 + Drizzle + OpenAuth
packages/slack · storybookSlack 集成 · UI 组件库

核心包技术栈(packages/opencode 依赖)

🧠模型层

Vercel AI SDK v5(ai)+ 近 20 个 provider 包:anthropic、openai、google、google-vertex、amazon-bedrock、azure、groq、mistral、cohere、xai、cerebras、deepinfra、togetherai、perplexity、alibaba、gateway、openai-compatible + openrouter…

🔗协议集成

@modelcontextprotocol/sdk(MCP)、@agentclientprotocol/sdk(ACP)。

🛠️代码智能

vscode-jsonrpc+vscode-languageserver-types(LSP);web-tree-sitter+tree-sitter-bash/powershell(解析 shell 做权限判定)。

⚙️运行时基建

effect(Effect-TS)+ OpenTelemetry;drizzle-orm(存储);node-pty/bun-pty(PTY);chokidar(监听);hono+zod(服务端);ulid、gray-matter、immer。

平台条件导入:#db → storage/db.bun.ts | db.node.ts,#pty → pty/pty.bun.ts | pty.node.ts,对 Bun 与 Node 运行时分别适配。
03

服务端:Hono + OpenAPI + 事件总线

服务端用 Hono + Zod 构建,响应类型由 Zod 推导 → OpenAPI 规格 → 生成 SDK。事件总线是「服务端跑、多端实时同步」的神经。

🌐API 表面(节选)

/session · fork/abort/share/summarize/revert/session/:id/message · prompt_async · command · shell/find · /find/file · /find/symbol/file/content · /file/status/lsp · /formatter · /mcp/agent · /command · /config/tui/* · /tui/control/event (SSE)

📡事件总线

GET /event 是 SSE 流,首事件 server.connected,之后推送内部 bus 事件(会话/消息/工具/权限/LSP/文件/TUI)。所有客户端据此实时同步同一份会话状态。

OpenAPI 规格在 /doc(如 localhost:4096/doc),可生成客户端或在 Swagger 里查看。

04

会话与消息数据模型

远比「消息数组」丰富——这是 OpenCode 编辑器级体验(分叉、回退、diff、子会话)的基础。

🗂️Session

可建/列/删/改标题;parentID 形成子会话(子代理工作发生在此);含 status、todo、diff。

fork 分叉revert / unrevertsummarize / initshare / unshareabortchildren

🧩Message + Part

一条消息由多个 Part 组成(文本/工具调用/工具结果/推理/文件附件),返回 { info, parts[] }。

发送三姿势:message(等待)、prompt_async(异步 204)、noReply:true(只注入上下文不回复);另有 command 与 shell。

结构化输出:prompt 传 format:{type:"json_schema", schema, retryCount},模型经内部 StructuredOutput 工具产出并校验 JSON,失败超限返回 StructuredOutputError。持久化经 #db(Drizzle ORM,Bun/Node 双实现),存于 ~/.local/share/opencode/。
05

Agent 主循环与模型层(AI SDK)

OpenCode 不自造模型调用与工具循环,而是构建在 Vercel AI SDK v5 之上,在其上叠加会话/权限/事件/快照编排。

图 2 · 单轮主循环(AI SDK 工具循环 + OpenCode 编排)

🎯 模型解析

启动优先级:--model → 配置 model → 上次使用 → 内部优先级首个。ID 为 provider/model。

📚 Provider 注册表

Models.dev(75+ provider 元数据)+ AI SDK 实现;enabled/disabled_providers 白黑名单(disabled 优先);small_model 跑轻量任务。

🎛️ Variants 变体

同模型不同推理档位(Anthropic high/max、OpenAI minimal→xhigh、Google low/high),variant_cycle 快速切换。

隐藏系统 agent参与主循环之外的自动化:compaction(压缩)、title(标题)、summary(摘要)。Provider 选项:timeout(默认 300s)、chunkTimeout、setCacheKey;模型级 thinking.budgetTokens、reasoningEffort 等可透传。

模型在代码库里「动手」的手段。默认全部启用且无需批准,行为由权限控制。

07

权限与安全模型

作为「能改你代码、能跑你 shell」的 agent,权限是安全底座。每条规则解析为 allow / ask / deny,按模式匹配、最后命中者胜。

// opencode.json
{
  "permission": {
    "*": "ask",            // 兜底在前
    "bash": {              // 对象语法 = 按输入匹配
      "*": "ask",
      "git *": "allow",
      "rm *": "deny"
    },
    "edit": { "*": "deny" }
  }
}

🌳 tree-sitter bash 解析

bash 权限匹配的是解析后的命令(如 git status --porcelain)——用 tree-sitter-bash 把一行 shell 拆成结构化命令再比对,避免字符串匹配被绕过。

🛡️ 两个安全守卫

external_directory:触碰工作目录外路径时触发(默认 ask)。doom_loop:同一工具调用以相同输入连续重复 3 次时触发(默认 ask),防死循环。

默认值:多数权限默认 allow;doom_loop/external_directory 默认 ask;read 默认 allow 但 .env/*.env.* 默认 deny(.env.example 放行)。「ask」三结果:once / always(本会话放行工具建议的安全模式,如 git status*)/ reject。agent 可覆盖,企业可经 MDM 强制不可覆盖。
08

Agents 体系(primary / subagent)

把「不同任务用不同配置的 AI」做成一等概念。可 JSON(opencode.json)或 Markdown(.opencode/agents/*.md,frontmatter+正文)配置。

🎚️主 agent(Tab 切换)

build — 默认,全工具,标准开发。

plan — 受限分析,edit/bash 默认 ask,适合探索陌生代码库而不误改。

🧭子代理(@ 调用,跑在子会话)

general — 通用多步(除 todo 全工具),可并行。

explore — 快速只读探索,不可改文件。

scout — 只读外部文档/依赖研究(克隆依赖进托管缓存)。

隐藏系统 agent:compaction / title / summary,自动运行不可选。关键选项:mode(primary/subagent/all)、model、prompt、temperature、steps(最大迭代)、permission(含 permission.task 控制可调用哪些子代理)、hidden、color + 透传 provider 字段。子代理产生子会话,TUI 提供父子会话穿梭键;opencode agent create 交互式创建。
09

LSP 与 MCP 集成

🔬LSP(代码诊断作反馈)

基于 vscode-jsonrpc。约 35 个内置语言服务器(typescript、pyright、gopls、rust-analyzer、clangd、jdtls、ruby-lsp、eslint、oxlint、svelte、vue、astro、zls…),按需自动安装。

默认关闭;启用后打开匹配文件即起服务器,把诊断回灌 agent loop。可 lsp:true 全开或对象语法逐个配置。

务实建议:LSP 较重,很多项目让 agent 直接跑 lint/typecheck CLI(写进 AGENTS.md)更划算。

🔌MCP(外部工具)

基于 @modelcontextprotocol/sdk。local(command+env)与 remote(url+headers),工具自动与内置并列。

remote OAuth 自动处理:检测 401→发起流程,支持动态客户端注册(RFC 7591),token 存 ~/.local/share/opencode/mcp-auth.json。

工具以「服务器名_」前缀注册;可「全局关、按 agent 开」控制上下文膨胀;组织可经 .well-known/opencode 远程下发。

10

上下文压缩与快照(undo/redo)

🗜️压缩 compaction

配置 compaction:auto(将满时自动压缩)、prune(移除旧工具输出省 token)、reserved(预留 token 缓冲)。由隐藏 compaction agent 生成续写摘要,/compact 手动触发。

插件钩子 experimental.session.compacting 可注入额外上下文(output.context)或完全替换压缩提示词(output.prompt)。

📸快照与回退

用内部影子 git 仓库跟踪 agent 改动(默认启用,可 snapshot:false 关)。

/undo 撤销最近用户消息、其后响应及文件改动;/redo 恢复;服务端 revert/unrevert 与按 messageID 的 diff 同样依赖快照。

因此 undo/redo 要求项目是 git 仓库。watcher.ignore 控制监听忽略模式。

11

扩展性:插件 / 钩子 / 命令 / Rules

扩展性建立在「插件 + 事件钩子 + SDK 客户端」之上。插件是导出插件函数的 JS/TS 模块,接收 { project, directory, worktree, client, $ },返回钩子对象。

▸核心钩子@opencode-ai/plugin

tool.execute.before / tool.execute.after — 执行前后改写参数/拦截(净化 bash、禁读 .env)。

event — catch-all,订阅事件总线。shell.env — 向所有 shell 注入环境变量。

experimental.session.compacting — 定制压缩。tool:{ mytool: tool({...}) } — 注册自定义工具(Zod schema,同名覆盖内置)。

export const EnvProtection = async () => ({
  "tool.execute.before": async (input, output) => {
    if (input.tool === "read" && output.args.filePath.includes(".env"))
      throw new Error("Do not read .env files")
  }
})
▸事件总线(钩子可订阅)events
session.created/compacted/deleted/idle/error/updatedmessage.updated/removedtool.execute.before/afterpermission.asked/repliedfile.editedfile.watcher.updatedlsp.updated / lsp.client.diagnosticstodo.updatedcommand.executedserver.connectedtui.prompt.append / command.execute / toast.show

加载顺序:全局配置 → 项目配置 → ~/.config/opencode/plugins/ → .opencode/plugins/;npm 插件启动时用 Bun 装到 ~/.cache/opencode/node_modules/。日志走 client.app.log()。

▸自定义命令 + Rulescommands · AGENTS.md

命令:command.<name> 或 .opencode/commands/*.md,字段 template(支持 $ARGUMENTS)、description、agent、model。

Rules:/init 分析项目生成 AGENTS.md(建议提交 git);instructions 配置可追加任意规则文件/glob,每次对话作为上下文塑造行为。

12

客户端矩阵与云协作

客户端/服务端解耦带来一整排客户端,全部经同一套 HTTP API。

⌨️ TUI

OpenTUI(SolidJS 终端渲染)。@ 引用文件、! 跑 shell、/ 命令、Tab 切 agent、leader 键 ctrl+x;独立 tui.json 配主题/keybinds/通知。

🖥️ Web / Desktop

Web(SolidStart,opencode web);桌面(Tauri,可发系统通知)。

🧩 IDE / ACP

编辑器扩展经 /tui 控制端点驱动;ACP(@agentclientprotocol/sdk)暴露为 Zed 等编辑器原生 agent。

🤖 SDK / CI

createOpencode()(自带起服务端)或 createOpencodeClient({baseUrl});GitHub/GitLab CI 集成。

🔗 Share(协作)

/share 生成公开链接 opncd.ai/s/<id>,同步会话到云;manual/auto/disabled;企业可禁用/限 SSO/自托管。云端:Cloudflare+S3+Drizzle+OpenAuth。

💲 Zen / Go

Zen:官方精选模型目录。Go:低价订阅($5 首月/$10 月)接入优质开源模型(GLM/Kimi/MiniMax/Qwen/DeepSeek),作普通 provider,完全可选无锁定。

13

配置体系与设计原则

配置 opencode.json/jsonc(TUI 用 tui.json),合并而非替换,8 级优先级(后者覆盖前者)。

1 远程配置 .well-known/opencode 组织默认（最低） 2 全局 ~/.config/opencode/opencode.json 3 OPENCODE_CONFIG（自定义路径） 4 项目 opencode.json 5 .opencode/ 目录（agents/commands/plugins/skills/tools…） 6 OPENCODE_CONFIG_CONTENT（运行时内联） 7 文件式 managed（/etc/opencode、%ProgramData%…） 8 macOS MDM 托管偏好 .mobileconfig 最高 · 用户不可覆盖
图 3 · 8 级配置优先级

变量替换:{env:VAR} / {file:path}。其它开关:autoupdate(true/notify/false)、formatter、shell、attachment.image(图片缩放)、experimental.policies(如 provider.use 的 allow/deny)。企业可经 Jamf/FleetDM 强制 permission/share/server。

横切设计原则

📐 API 优先

一切经 OpenAPI 暴露,SDK 自动生成,客户端与内核以类型契约解耦。

⌨️ 本地服务化

agent 是本地无头服务,可被任意 UI/脚本/CI 编排。

📡 事件驱动

内部 bus → SSE,多端实时同步同一会话。

🔒 可审计安全

细粒度权限 + tree-sitter bash 解析 + external_directory/doom_loop 守卫 + .env 默认拒读。

↩️ Git 原生可回退

影子 git 快照让每步改动可 undo/redo/revert。

🧩 模型无关

AI SDK + Models.dev,75+ provider、统一 ID、变体、无锁定。

总评

OpenCode 的架构可概括为「一个本地无头服务内核 + 一圈经 OpenAPI 解耦的客户端 + 一套 git 原生、权限受控的编码工作流」。客户端/服务端 + OpenAPI 是地基,让 TUI/Web/桌面/IDE/ACP/SDK/CI 共享同一内核;它构建在成熟生态之上(AI SDK 处理模型、MCP/ACP 处理协议、LSP/tree-sitter 处理代码智能、Drizzle/影子 git 处理状态与回退),自身专注「会话/工具/权限/事件」的编排。最值得借鉴:用 OpenAPI 把 agent 内核变成可被任意客户端消费的本地服务、用事件总线驱动多端同步、用 tree-sitter 把 shell 权限升级为结构化匹配、用影子 git 实现可靠的改动回退。
