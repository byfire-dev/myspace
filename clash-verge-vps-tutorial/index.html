Principle 路 防漏损原理
为什么系统代理不够安全
LAYER 3

在开发环境中，高频 AI 工具、命令行、包管理器和脚本请求并不总是遵守浏览器代理设置。真正要避免网络身份漂移，必须把代理从“应用层设置”下沉到“系统路由层”。

传统系统代理的边界

传统 Shadowsocks 客户端的“全局模式”通常只是修改系统的 HTTP/HTTPS 代理设置。浏览器、部分桌面应用可以识别这些设置，但终端、Git、Node.js、curl、Python 脚本并不一定会自动继承。

核心风险：浏览器显示海外 IP，但本地终端或 AI CLI 工具仍然用真实网络直连，导致同一账号出现跨区域访问轨迹。

TUN 模式的工作方式

Clash Verge Rev 的 TUN 模式会创建一张系统级虚拟网卡，把离机前的三层 IP 流量接管到 Clash 内核，再按规则分流到代理节点或直连出口。

关键收益：不依赖单个应用是否支持代理配置，浏览器、终端、开发工具链和 CLI 请求都能被统一纳入同一套出口策略。

应用层代理 vs 路由层接管
维度 系统 HTTP 代理 Clash TUN 模式
作用层级 应用层，由应用主动读取代理设置 网络层，由系统路由表把流量送入虚拟网卡
覆盖对象 浏览器、部分桌面软件 浏览器、终端、CLI、后台脚本、包管理器
典型风险 终端绕过代理，账号环境出现真实 IP 配置错误时可能影响 DNS 或局域网访问，需要验证
适用场景 轻量网页浏览 开发者日常、AI CLI、统一出口身份控制

判断代理是否可靠，不看浏览器页面能不能打开，而看 PowerShell 里的 curl https://ipinfo.io 是否返回同一个海外 VPS 出口。

Mapping 路 参数提取
把 VPS 节点翻译成 Clash 字段
YAML

Clash 核心读取的是 YAML 配置。你需要先从 VPS 面板或原 Shadowsocks 客户端中拿到四个关键参数，再一一映射到 Clash 的 proxies 节点定义里。

原始信息 你需要确认什么 Clash YAML 字段
Server VPS 公网 IP 或域名 server: 67.216.195.130
Port 服务端开放的入站端口 port: 46123
Password 节点鉴权密码 password: "YourSecurePassword"
Cipher 加密算法，例如 aes-256-gcm cipher: chacha20-ietf-poly1305
01
先确认节点可用

不要急着写 Clash 配置，先确认原客户端能正常连接，密码、端口和加密方式没有过期或填错。

02
再写最小配置

先只保留一个节点、一个选择组和基础规则，跑通后再加复杂分流，降低排错成本。

03
最后开启 TUN

普通代理可用后再开启 TUN。若一开始就打开系统级接管，配置错误会让问题更难定位。

YAML 规则：缩进只能用空格，不能用 Tab；冒号后必须有空格；节点字段必须缩进在 proxies 列表项下面，不能散落在顶层。

可复用判断：如果 Clash 报语法错误，优先检查缩进层级；如果能启动但不能连接，优先检查端口、密码、加密方式和服务端防火墙。

Profile 路 标准配置
可直接改写的本地 YAML 模板
PROFILE

下面是一份单节点 Shadowsocks 模板。把示例 IP、端口、密码和加密方式换成你自己的 VPS 信息后，再导入 Clash Verge Rev。

# Profile Template for Clash Verge Rev mixed-port: 7890 allow-lan: false mode: rule log-level: info external-controller: 127.0.0.1:9090 # 1. 节点基础定义 proxies: - name: "自建 VPS" type: ss server: 67.216.195.130 # 改成你的实际 VPS 公网 IP port: 46123 # 改成你的实际端口 cipher: chacha20-ietf-poly1305 # 按实际加密方式修改 password: "YourSecurePassword" # 改成你的连接密码 udp: true # 2. 策略组控制 proxy-groups: - name: "节点选择" type: select proxies: - "自建 VPS" - DIRECT # 3. 基础及 Claude 相关分流规则 rules: - DOMAIN-SUFFIX,anthropic.com,节点选择 - DOMAIN-SUFFIX,claude.ai,节点选择 - DOMAIN-KEYWORD,claude,节点选择 - GEOIP,CN,DIRECT - MATCH,节点选择

推荐保存方式：在 Clash Verge Rev 中创建 Local Profile 后直接编辑保存，避免系统文本编辑器自动改后缀或编码。

不要照抄示例：67.216.195.130、46123 和 YourSecurePassword 都只是占位内容，必须替换。

Operation 路 软件实操
在 Clash Verge Rev 中导入本地配置
LOCAL

导入动作本身很短，但要把“创建、替换、保存、激活”四个步骤一次做完整。任何一步漏掉，Clash 内核都可能仍在使用旧配置。

01
新建 Local Profile

打开 Clash Verge Rev，进入 Profiles，点击 Create，把 Type 设为 Local，Name 可填写 MyVPS。

02
清空默认内容

右键新生成的配置卡片，选择 Edit。先用 Ctrl + A 全选，再删除原有模板内容。

03
粘贴并保存

粘贴第 03 页的模板，替换自己的节点参数，点击 Save。保存后回到 Profiles 页面确认卡片存在。

激活与初步排错
动作 成功信号 如果失败，先查什么
激活配置卡片 卡片出现选中态或高亮边框 是否保存到正确的 Local Profile
打开代理开关 系统代理端口可用，浏览器可访问目标站点 Clash 日志是否提示 YAML 解析错误
选择节点 策略组中的“自建 VPS”处于选中状态 节点名称是否与规则、策略组完全一致

操作顺序：先让普通代理跑通，再开启 TUN。这样可以把问题拆成两层：节点是否可用，以及系统级接管是否生效。

TUN Mode 路 底层接管
开启 TUN 并验证终端出口
VERIFY

这是让 Claude Code、curl、Git、Node.js 等开发工具链不绕过代理的关键步骤。验证必须在终端里完成，而不是只看浏览器。

开启 TUN 模式

  • 进入 Settings：在 Clash Verge Rev 左侧导航打开设置页。
  • 打开 TUN Mode：找到 TUN Mode 开关并启用。
  • 授权系统权限：Windows 首次开启时会弹出 UAC 管理员权限申请，选择允许。
  • 等待虚拟网卡就绪：开启后稍等数秒，再进行终端验证。

终端验证命令

打开 PowerShell、VS Code Terminal 或 macOS Terminal，执行：

curl https://ipinfo.io

返回结果里的 ip 应该是你的 VPS 出口 IP，country 应该与 VPS 所在地区一致。

验证标准
结果 含义 下一步
返回 VPS IP 终端流量已被 TUN 接管并走代理 可以继续使用 AI CLI、Git、curl 等开发工具
返回本地真实 IP TUN 未生效，终端仍在直连 检查 TUN 开关、系统权限、Clash 日志和路由规则
请求超时或 DNS 错误 可能是 DNS、虚拟网卡或防火墙冲突 关闭 TUN 回退验证，再逐项排查 DNS 与安全软件

重要提醒：不要用“浏览器能访问”替代终端验证。开发工具链里的风险，只有在真实命令行请求里才能看见。

Checklist 路 使用与维护
把配置变成稳定日常流程
RUNBOOK

一次配置成功不代表长期稳定。节点、系统权限、Clash 版本和安全软件都可能变化，建议把下面的检查清单作为日常 runbook。

每次启动 AI CLI 前

  • 确认 Clash Verge 已启动：代理开关和策略组处于正确状态。
  • 确认 TUN 仍开启：系统更新或软件重启后可能需要重新授权。
  • 执行出口测试：运行 curl https://ipinfo.io，确认终端出口是 VPS。
  • 再启动工具：出口验证通过后，再运行 Claude Code 或其他 AI CLI。

常见问题定位

症状 优先检查
配置无法保存 YAML 缩进、冒号空格、引号闭合
节点不可用 IP、端口、密码、cipher、服务端状态
终端不走代理 TUN 开关、管理员权限、虚拟网卡
国内网站异常 规则顺序、GEOIP 数据库、DNS 设置

最终标准：浏览器、终端和 AI CLI 的出口 IP 必须一致。只要任一入口显示不同网络身份，就还不能视为“防漏损配置完成”。
