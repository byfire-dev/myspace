Training Platform Blueprint
THE LOOP
从直播获客到作品交付

平台的核心不是播放课程，而是让每个用户从线索、成交、学习、实操到复购都被系统承接。

1
Acquire

社媒内容、直播预约、试听课和表单进入轻量 CRM，记录渠道与转化路径。

2
Learn

课程不只是视频目录，而是“短视频 + 讲义 + 模板 + Agent 入口 + 作业”。

3
Build

学员在垂直 Agent 工作台里完成账号方案、代码项目、小说设定或短剧脚本。

4
Review

AI 初评、导师复评、评分 Rubric 和修改记录共同形成可追踪的交付标准。

5
Compound

优秀案例、失败样例、提示词和流程回写到知识库，下一期训练营更好交付。

CORE SYSTEM
它不是网校，而是交付操作系统

真正要自研的是课程资产、Agent 资产、案例资产和学员数据资产的沉淀系统。

Course Assets

课程、讲义、模板、直播回放、阶段项目和结营标准。

Agent Assets

垂直 Agent、Prompt 版本、工作流、工具调用和安全规则。

Project Assets

作业、作品集、优秀案例、反例和导师点评数据。

Growth Assets

渠道来源、直播转化、学习留存、复购路径和企业客户线索。

WRONG FRAME
只做“视频 + 聊天框”会很快失去壁垒

答案本身会过期，真正能复利的是标准、案例、流程、评价和用户行为数据。

FIRST THREE

ROADMAP
先跑通交付，再做平台生态

第一版目标不是“大而全”，而是验证训练营交付闭环和 Agent 对学习成果的提升。

0—3个月

课程、作业、回放、3个 Agent、基础后台和数据看板。

3—6个月

班级管理、助教工作台、AI初评、CRM和会员额度。

6—12个月

Agent 版本管理、企业空间、团队看板和安全审核体系。

12个月后

Agent 市场、讲师入驻、企业私有 Agent 和行业方案。

先用一门课验证交付，再把 Agent 产品化，最后复制到更多垂直课程。

不要优先自研直播推流、自研大模型、复杂 IM 或 Agent 市场。先把学习、实操、作业和复购跑顺。
