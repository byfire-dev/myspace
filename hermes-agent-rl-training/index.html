RL TRAINING

一条开源 AI 助手自我进化流水线：让工具调用轨迹先被记录，再被压缩，最后进入可打分、可更新、可评测的强化学习环境。

CORE IDEA 训练工具型 Agent，关键样本不是单轮问答，而是完整的多轮决策轨迹。 SOURCE 整理自知乎文章《Hermes Agent 的内置 RL 训练：一条让开源 AI 助手自我进化的流水线》。 › Open Article
把“会调用工具”变成“能通过环境反馈持续变强”。

Hermes 的工程重点不是单独训练一个模型，而是把日常对话、轨迹沉淀、token 预算控制、RL rollout 和基准评测组织成同一套系统。

Why Trajectory

工具调用不是“问答”，而是环境交互

普通 SFT 的问答样本只能教模型给出答案，很难教会模型何时停、该选哪个工具、工具返回后如何继续决策。轨迹数据把这些隐藏行为显式记录下来。

1
保存完整会话

成功样本进入 `trajectory_samples.jsonl`，失败样本进入 `failed_trajectories.jsonl`，便于训练和复盘。

2
统一 ShareGPT 格式

把 assistant、tool_calls、tool_response 和最终回答归一成训练数据集常用结构。

3
保留 reasoning 字段

多 provider 的 reasoning content 被提取和恢复，用于支持会思考再调用工具的 chat template。

agent conversation -> save_trajectory() -> ShareGPT conversations -> compressed.jsonl -> Atropos rollout + reward
Compression

压缩的难点：不能把关键决策截掉

一次 SWE 任务可能包含几十轮工具调用，原始上下文会轻易超过训练预算。Hermes 的策略是保护关键两端，中间转成摘要，而不是粗暴截断。

保护开头

system、human、first gpt、first tool 保留任务目标和初始路径。

保护结尾

最后若干轮通常包含修复、测试和收束判断。

压缩中段

探索、搜索、失败尝试被总结成短摘要。

显式提示

训练样本告诉模型：部分历史工具结果已被摘要化。

这一步的工程价值在于：让长轨迹仍然能进入 16K 左右的训练窗口，同时尽量保留“为什么这么做”的行为证据。

RL Environment

Atropos 让 Agent 行为可打分

`HermesAgentBaseEnv` 把 Atropos 训练循环接入 Hermes 的 agent loop。模型在环境中调用 Terminal、File、Search、Browser 等工具，reward 函数直接检查沙盒状态。

A
Phase 1

用 OpenAI 兼容接口跑评测或生成 SFT 数据，依赖服务端解析 tool calls。

B
Phase 2

用 vLLM ManagedServer 拿到精确 token ids 和 logprobs，客户端解析工具调用，进入 GRPO 或 PPO。

compute_reward(ctx) -> ctx.terminal("pytest -v") -> ctx.read_file("solution.py") -> score rollout behavior
Path A

从零构建训练数据

先用 Hermes Agent 批量跑任务收集轨迹，再用 `sample_and_compress.py` 采样压缩，最后进入 SFT 或后续 RL。

Path B

直接接入 Atropos

准备 vLLM、SGLang 或 OpenRouter 这类 API server，启动 Hermes SWE 环境，由 rollout 与 reward 驱动训练。

Path C

评测已有模型

在 TerminalBench 2.0 或 SWE 风格任务中验证模型是否真的能读文件、改代码、跑测试并收敛到正确结果。

这套系统的核心价值：训练的不是“答案风格”，而是“工具行为策略”。

它把开源 Agent 的使用过程变成训练资产。日常对话可以产生轨迹，失败案例可以变成诊断样本，评测环境可以把“有没有真正完成任务”转成可优化的奖励。

Engineering Notes
Sandbox
每个 rollout 独立 task_id，文件和进程状态隔离，但 reward 函数仍可读取结果。
Async
工具调用放进 ThreadPoolExecutor，避开 Modal、Docker、Daytona 内部 event loop 冲突。
Parser
支持 Hermes、Mistral、Qwen、DeepSeek、Kimi、GLM 等多种 tool call 格式。
Database
SQLite 高并发写入通过随机 jitter 重试和 WAL checkpoint 降低锁竞争。
