01 / Objective

目标与前置条件

目标是在 Linux VPS 上安装并运行 Shadowsocks-rust 服务端，让服务以 systemd 托管并开机自启。

服务端程序 ssserver，部署到 /usr/local/bin/ssserver
配置目录 /etc/shadowsocks-rust，主配置为 config.json
系统服务 shadowsocks-rust.service，由 systemd 管理
✓已经登录搬瓦工 VPS。
✓当前用户为 root，或具备 sudo 权限。
✓系统支持 systemd。
✓已下载 Shadowsocks-rust 安装包。
PackageLinux x86_64
shadowsocks-v1.24.0.x86_64-unknown-linux-gnu.tar.xz
安全提醒：文档中的密码 123456 仅作为示例。生产环境必须替换为高强度随机密码。
02 / Install

安装 Shadowsocks-rust

进入安装包所在目录，先解压二进制包，再把 ssserver 放到系统可执行路径。

Extracttar
tar -xf shadowsocks-v1.24.0.x86_64-unknown-linux-gnu.tar.xz
Install Binary/usr/local/bin
cp ssserver /usr/local/bin/ssserver
chmod +x /usr/local/bin/ssserver
Verifyversion
/usr/local/bin/ssserver --version
03 / Configuration

创建配置文件与 systemd 服务

Shadowsocks-rust 的部署核心是两份文件：服务配置 config.json 与 systemd unit 文件。

创建配置目录

Directoryconfig home
mkdir -p /etc/shadowsocks-rust

创建 Shadowsocks 配置文件

Editconfig.json
vi /etc/shadowsocks-rust/config.json
JSON/etc/shadowsocks-rust/config.json
{
  "server": "0.0.0.0",
  "server_port": 8443,
  "password": "123456",
  "timeout": 300,
  "method": "aes-256-gcm",
  "fast_open": true
}
字段 说明
server 服务监听地址，0.0.0.0 表示监听所有网卡。
server_port 服务端口，此处为 8443。
password 客户端连接密码，实际部署时应替换为强密码。
timeout 连接超时时间，单位为秒。
method 加密方式，此处为 aes-256-gcm。
fast_open 是否启用 TCP Fast Open。

创建 systemd 服务

Editservice unit
vi /etc/systemd/system/shadowsocks-rust.service
Unit File/etc/systemd/system/shadowsocks-rust.service
[Unit]
Description=Shadowsocks-rust Service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/ssserver -c /etc/shadowsocks-rust/config.json
Restart=on-failure

[Install]
WantedBy=multi-user.target
04 / Service

启动并验证服务状态

创建或修改 service 文件后，必须先让 systemd 重新加载配置，再启动服务。

Reloadsystemd
systemctl daemon-reload

启用并立即启动

Enableauto start
systemctl enable shadowsocks-rust --now

只手动启动一次

Startmanual
systemctl start shadowsocks-rust

查看服务状态与日志

Statusactive running
systemctl status shadowsocks-rust

正常情况下，应能看到服务处于 active (running) 状态。需要进一步排查时查看日志。

Logsrecent 100 lines
journalctl -u shadowsocks-rust -n 100 --no-pager
Logsfollow
journalctl -u shadowsocks-rust -f
05 / Network

端口监听、防火墙与终端语言环境

服务启动后，还需要确认端口真的监听，并且系统防火墙、搬瓦工控制面板或上游服务商的防火墙规则都放行了该端口。

检查端口监听

Socketss
ss -lntp | grep 8443

如果系统没有 ss 命令，可以使用 netstat。

Socketnetstat fallback
netstat -lntp | grep 8443

放行防火墙端口

ufw

Firewallufw
ufw allow 8443/tcp
ufw reload

firewalld

Firewallfirewalld
firewall-cmd --permanent --add-port=8443/tcp
firewall-cmd --reload

终端语言环境配置

如果执行 top 等命令时出现语言、字符集或显示异常，可以临时设置语言环境。

Localebasic

export LANGUAGE="en_US"

06 / Automation

AI 自动化配置参考命令

以下命令适合 AI 或自动化脚本参考。执行前必须替换 SS_PASSWORD。

Scriptreplace password before run
SS_PORT="8443"
SS_PASSWORD="CHANGE_ME_TO_A_STRONG_PASSWORD"
SS_METHOD="aes-256-gcm"
SS_CONFIG_DIR="/etc/shadowsocks-rust"
SS_CONFIG_FILE="${SS_CONFIG_DIR}/config.json"
SS_SERVICE_FILE="/etc/systemd/system/shadowsocks-rust.service"

mkdir -p "${SS_CONFIG_DIR}"

cat > "${SS_CONFIG_FILE}" <<EOF
{
  "server": "0.0.0.0",
  "server_port": ${SS_PORT},
  "password": "${SS_PASSWORD}",
  "timeout": 300,
  "method": "${SS_METHOD}",
  "fast_open": true
}
EOF

cat > "${SS_SERVICE_FILE}" <<EOF
[Unit]
Description=Shadowsocks-rust Service
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/ssserver -c ${SS_CONFIG_FILE}
Restart=on-failure

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable shadowsocks-rust --now
systemctl status shadowsocks-rust --no-pager

08 / Troubleshooting

常见问题排查

服务启动失败

先看 systemd 日志，再检查二进制、权限、JSON 与端口占用。

Logssystemd
journalctl -u shadowsocks-rust -n 100 --no-pager
  • /usr/local/bin/ssserver 是否存在。
  • ssserver 是否有执行权限。
  • /etc/shadowsocks-rust/config.json 是否是合法 JSON。
  • 8443 端口是否已被其他服务占用。

客户端无法连接

把问题拆成服务器、客户端参数、防火墙和运行状态四层。

  • VPS 公网 IP 是否正确。
  • 客户端端口、密码、加密方式是否与服务端一致。
  • VPS 系统防火墙是否放行 8443/tcp。
  • 搬瓦工控制面板或上游防火墙是否放行该端口。
  • 服务是否正在运行。
Statusno pager
systemctl status shadowsocks-rust --no-pager

修改配置后不生效

修改 config.json 后重启服务；如果修改的是 service 文件，需要先重新加载 systemd。

Config Changedrestart
systemctl restart shadowsocks-rust
Service Changedreload then restart
systemctl daemon-reload
systemctl restart shadowsocks-rust
09 / Execution Order

推荐执行顺序汇总

完整操作可以压缩为以下顺序。实际执行时，不要跳过状态检查和防火墙确认。

  1. 解压 Shadowsocks-rust 安装包。
  2. 复制 ssserver 到 /usr/local/bin 并授予执行权限。
  3. 创建 /etc/shadowsocks-rust 配置目录。
  4. 写入 config.json 与 shadowsocks-rust.service。
  5. 执行 systemctl daemon-reload。
  6. 执行 systemctl enable shadowsocks-rust --now。
  7. 查看服务状态并确认 8443/tcp 已监听。

tar -xf shadowsocks-v1.24.0.x86_64-unknown-linux-gnu.tar.xz
cp ssserver /usr/local/bin/ssserver
chmod +x /usr/local/bin/ssserver

mkdir -p /etc/shadowsocks-rust
vi /etc/shadowsocks-rust/config.json
vi /etc/systemd/system/shadowsocks-rust.service

systemctl daemon-reload
